A. JavaScript 入門

  • A1. JavaScript 基本概念介紹
    JS 是什麼？
    • JavaScript (簡稱 JS) 是一種高階、直譯式的程式語言，專門為網頁開發而設計。
    • 它讓靜態的 HTML 網頁變得動態且具有互動性，是現代網頁開發不可或缺的技術。
    • JavaScript 於 1995 年由 Brendan Eich 在 Netscape 公司開發，最初只用了 10 天時間建立。
    • 現在已成為世界上最流行的程式語言之一，不僅用於前端，也廣泛應用於後端開發。
    JavaScript 的特色
    • 直譯式語言：不需要編譯，瀏覽器直接執行
    • 動態型別：變數型別在執行時決定
    • 跨平台：可在各種作業系統上執行
    • 物件導向：支援物件導向程式設計
    • 函式式程式設計：函式是第一級物件
    • 事件驅動：基於事件的程式設計模式
    JavaScript 用在哪裡？
    前端開發 (Frontend)
    • 網頁互動：按鈕點擊、表單驗證、動畫效果
    • 單頁應用 (SPA)：React, Vue.js, Angular
    • 使用者介面：動態內容更新、即時回饋
    • 瀏覽器 API：地理位置、相機、通知等
    後端開發 (Backend)
    • Node.js：伺服器端 JavaScript 執行環境
    • API 開發：RESTful API、GraphQL
    • 資料庫操作：MongoDB、MySQL 等
    • 即時應用：聊天室、線上遊戲
    行動應用開發
    • React Native：跨平台行動應用
    • Ionic：混合式行動應用
    • PWA：漸進式網頁應用程式
    • Cordova/PhoneGap：混合應用開發
    其他應用領域
    • 桌面應用：Electron (VS Code, Discord)
    • 遊戲開發：網頁遊戲、遊戲引擎
    • 人工智慧：TensorFlow.js 機器學習
    • IoT 物聯網：智慧家居、感測器控制
    與 HTML/CSS 的關係
    網頁開發三劍客
    HTML

    結構層 (Structure)
    定義網頁的內容和結構
    如：標題、段落、圖片、連結

    CSS

    表現層 (Presentation)
    控制網頁的外觀和樣式
    如：顏色、字型、版面配置

    JavaScript

    行為層 (Behavior)
    賦予網頁互動功能和動態效果
    如：事件處理、資料處理、動畫

    三者協作範例

    想像一個簡單的登入表單：

    HTML 負責
    • 建立表單元素
    • 輸入欄位 (帳號、密碼)
    • 提交按鈕
    • 基本結構
    CSS 負責
    • 表單的外觀美化
    • 輸入框樣式
    • 按鈕顏色和效果
    • 版面配置
    JavaScript 負責
    • 驗證輸入格式
    • 處理按鈕點擊事件
    • 顯示錯誤訊息
    • 送出資料到伺服器
    重要觀念
    • 漸進增強 (Progressive Enhancement)：先有 HTML 基礎結構，再用 CSS 美化，最後用 JavaScript 增加互動功能
    • 分離關注點 (Separation of Concerns)：結構、樣式、行為各司其職，便於維護和修改
    • 優雅降級 (Graceful Degradation)：即使 JavaScript 被停用，網頁基本功能仍可正常運作
  • A2. 如何在 HTML 中使用 JavaScript
    <script> 標籤
    三種引入方式

    JavaScript 可以透過三種方式加入到 HTML 中：

    內嵌式 (Inline)

    直接寫在 HTML 元素的事件屬性中

    <button onclick="alert('你好！')">
  點我
</button>
    內部式 (Internal)

    寫在 <script> 標籤內部

    <script>
function sayHello() {
  alert('你好！');
}
</script>
    外部式 (External)

    引入外部 .js 檔案

    <script src="script.js"></script>
    1. 內嵌式 JavaScript

    優點：簡單快速，適合測試小片段程式碼

    缺點：不易維護，違反分離關注點原則

    範例：
    <!-- 按鈕點擊事件 -->
<button onclick="alert('歡迎來到我們的網站！')">歡迎訊息</button>

<!-- 滑鼠移入事件 -->
<img src="photo.jpg" 
     onmouseover="this.style.opacity='0.8'" 
     onmouseout="this.style.opacity='1'">

<!-- 表單提交驗證 -->
<form onsubmit="return confirm('確定要提交嗎？')">
  <input type="text" placeholder="輸入姓名">
  <button type="submit">提交</button>
</form>
    2. 內部式 JavaScript

    優點：集中管理，比內嵌式更好維護

    缺點：HTML 檔案變大，無法跨頁面重複使用

    <script> 標籤位置很重要：
    ❌ 放在 <head> 中：
    <head>
  <script>
    // DOM 元素還未載入
    document.getElementById('btn')
      .onclick = function() {
        alert('點擊！');
      };
  </script>
</head>
<body>
  <button id="btn">按鈕</button>
</body>
    ✅ 放在 </body> 前：
    <body>
  <button id="btn">按鈕</button>
  
  <script>
    // DOM 元素已載入
    document.getElementById('btn')
      .onclick = function() {
        alert('點擊！');
      };
  </script>
</body>
    3. 外部式 JavaScript (推薦)

    優點：分離關注點、可重複使用、易於維護、瀏覽器可快取

    缺點：需要額外的 HTTP 請求

// 進階版：支援多種運算
function advancedCalculate(operation) {
  const num1 = parseFloat(document.getElementById('num1').value) || 0;
  const num2 = parseFloat(document.getElementById('num2').value) || 0;
  let result;
  
  switch(operation) {
    case 'add': result = num1 + num2; break;
    case 'subtract': result = num1 - num2; break;
    case 'multiply': result = num1 * num2; break;
    case 'divide': result = num2 !== 0 ? num1 / num2 : '除數不能為零'; break;
    default: result = 0;
  }
  
  document.getElementById('calcResult').innerHTML = `結果：${result}`;
}

    function addItem() {

    JavaScript 基本符號說明

    在開始學習變數宣告之前，讓我們先認識 JavaScript 程式碼中的基本符號，這些符號在接下來的學習中會經常用到：

    符號 名稱 用途
    ; 分號 程式碼行結尾
    = 等號 賦值（給變數設定值）
    " ' 引號 包圍文字內容
    ( ) 小括號 運算優先序
    // 雙斜線 寫註解說明
    + - * / 運算符號 數學運算
    📝 暫時不用記： 其他符號如 { }、[ ]、.、=== 等，我們會在後面的章節中詳細介紹。
    🔤 基本語法說明
    1. 什麼是程式碼行？

    程式碼行就像是給電腦的一個指令：

    let name = "小明";  // 這是一行指令
    2. 註解的寫法

    註解是給自己看的說明：

    // 這是註解，電腦會忽略這行
let age = 25;  // 這也是註解
    3. 小括號改變順序

    就像數學運算一樣：

    5 + 3 * 2;    // 結果是 11 (先乘後加)
(5 + 3) * 2;  // 結果是 16 (先加後乘)
    4. 常見錯誤
    • 忘記分號：let name = "小明" ❌
    • 正確寫法：let name = "小明"; ✅
    • 引號不匹配：let name = "小明'; ❌
    • 正確寫法：let name = "小明"; ✅
    💡 提示：認識這些基本符號後，接下來我們就能更輕鬆地理解變數宣告的語法了！
    什麼是變數？為什麼需要變數？

    變數就像是一個「容器」或「標籤」，用來存放和管理資料。想像一下：

    生活中的例子
    • 置物盒：每個盒子都有標籤，裡面放不同的東西
    • 通訊錄：每個聯絡人都有名字，儲存電話號碼
    • 銀行帳戶：每個帳戶都有戶名，存放餘額
    程式碼中的例子
    • name：存放使用者的姓名
    • age：存放使用者的年齡
    • score：存放考試成績
    為什麼需要變數？
    儲存資料

    暫時保存使用者輸入、計算結果等資訊

    重複使用

    一次儲存，多次使用，避免重複計算

    動態修改

    可以隨時更新變數的值，讓程式更靈活

    實際範例對比
    沒有變數的寫法（不好）

3.14159 * 5 * 5 * 2;    // 計算兩個圓的面積
3.14159 * 5 * 5 / 4;    // 計算四分之一圓的面積

// 問題：重複、容易出錯、難以修改
    使用變數的寫法（好）
    // 使用變數存放資料

let radius = 5;

// 現在可以輕鬆重複使用

// 優點：清楚、易讀、易修改
    什麼是「宣告」？

    在程式設計中，「宣告」就是告訴電腦：「我要建立一個新的變數」。就像是在倉庫裡放置一個新的儲物盒，並且給它貼上標籤。

    宣告的過程
    1. 告訴電腦：「我要建立一個變數」
    2. 給它命名：「這個變數叫什麼名字」
    3. 分配記憶體：「電腦準備空間來存放資料」
    4. 可選擇賦值：「可以馬上放入資料，也可以稍後再放」
    // 1. 宣告變數（只建立，沒有賦值）
let userName;           // 只宣告，值是 undefined

// 2. 宣告並賦值（建立並馬上放入資料）
let userAge = 25;       // 宣告同時賦值

// 3. 後續賦值（稍後再放入資料）
userName = '小明';       // 賦值給之前宣告的變數
    生活中的類比

    📦 倉庫管理員的工作：

    • 宣告 = 「我要放一個新的盒子」
    • 命名 = 「這個盒子叫『使用者姓名』」
    • 分配空間 = 「找一個位置放盒子」
    • 賦值 = 「把東西放進盒子裡」
    // 就像倉庫管理：

使用者姓名 = '小明';      // 把資料放進盒子

// 或者一次完成：

    為什麼要宣告？

    • 📋 明確告知：讓電腦知道這個名稱是變數
    • 🔒 避免錯誤：防止使用不存在的變數
    • 💾 記憶體管理：電腦可以準備適當的存儲空間
    • 🎯 作用域控制：決定變數在哪裡可以使用

    沒有宣告會怎樣？

    JavaScript 中變數宣告的三種方式

    了解了什麼是宣告和宣告的步驟後，現在來看看 JavaScript 提供的三種宣告方式：let、const、var。每種方式都有不同的特性和用途：

    let（推薦）
    • 可以重新賦值
    • 區塊作用域
    • 不可重複宣告
    let name = '小明';
name = '小華';        // ✅ 可以修改
let name = '小美';    // ❌ 不能重複宣告
    const（常數）
    • 不可重新賦值
    • 區塊作用域
    • 不可重複宣告
    const PI = 3.14159;
PI = 3.14;           // ❌ 不能修改
const PI = 3.1416;   // ❌ 不能重複宣告
    var（避免使用）
    • 可以重新賦值
    • 函式作用域
    • 可以重複宣告
    var oldWay = '舊方式';
var oldWay = '重複';  // ✅ 但不建議
oldWay = '修改';     // ✅ 可以修改
    最佳實踐建議
    • 優先使用 const：如果值不會改變（如 PI、設定值）
    • 需要改變時用 let：如計數器、使用者輸入
    • 完全避免 var：舊語法，容易造成問題

    什麼是型別？

    在程式設計中，「型別」就是告訴電腦這個資料是什麼種類的。就像現實生活中，我們會把東西分類一樣：蘋果是水果、書本是文具、汽車是交通工具。

    生活中的分類
    • 水果：蘋果、香蕉、橘子
    • 數字：1、2、3.14、-10
    • 文字：「你好」、「小明」、「ABC」
    • 是非：對、錯、真、假
    程式中的分類
    • 數字型別：25、3.14、-100
    • 字串型別："你好"、"小明"、"ABC"
    • 布林型別：true、false
    • 陣列型別：[1, 2, 3]、["A", "B"]
    為什麼需要型別？
    1. 防止錯誤 
    let age = 25;
age + "歲";     // "25歲" ✓
age - "歲";     // NaN ✗
    2. 提供對應操作 
    let text = "Hello";
text.length;        // 5 ✓
text.toUpperCase(); // "HELLO" ✓
    3. 節省記憶體 
    let flag = true;    // 1位元組
let num = 42;       // 8位元組
let text = "Hi";    // 依長度而定
    型別的神奇之處

    JavaScript 是「動態型別」語言，意思是同一個變數可以存放不同型別的資料：

    let box = 42;           // 現在是數字
console.log(typeof box); // "number"

box = "Hello";          // 現在是字串
console.log(typeof box); // "string"

box = true;             // 現在是布林
console.log(typeof box); // "boolean"

    型別 說明 範例

    String 字串 "Hello", '世界'
    Boolean 布林值 true, false
    undefined 未定義 undefined
    null 空值 null

    參考型別 (Reference Types)

    基本型別 vs 參考型別的差異

    用圖書館借書系統來理解：基本型別像「影印書」，參考型別像「借書證」

    📚 圖書館借書系統統一比喻
    � 基本型別 = 影印書

    就像在圖書館影印整本書：

    • 你拿到書的完整副本
    • 可以在副本上做筆記
    • 不會影響原書
    • 每個人都有獨立的書
    🎫 參考型別 = 借書證

    就像拿到借書證（位置資訊）：

    • 你拿到書的位置資訊
    • 所有人去同一書架拿書
    • 任何人做筆記都會影響別人
    • 書只有一本，但借書證可以很多張
    基本型別（值型別）

    📄 影印書概念：拿到完整副本

    let bookA = "JavaScript入門";     // 影印書A
let bookB = bookA;                // 影印書B
bookA = "JavaScript進階";         // 書A改標題
console.log(bookB);               // "JavaScript入門" (書B沒變)
    📚 圖書館情境：
    • 你和朋友各自影印一本書
    • 你在書上做筆記，朋友看不到
    • 每個人都有獨立的書
    參考型別（Reference Type）

    🎫 借書證概念：拿到位置資訊

    let card1 = {book: "JavaScript入門"}; // 借書證1

card1.book = "JavaScript進階";        // 改變書架上的書
console.log(card2.book);              // "JavaScript進階" (同一本書!)
    📚 圖書館情境：
    • 你和朋友拿到同一個書架位置
    • 任何人做筆記，大家都看得到
    • 操作的是同一本書
    📚 圖書館情境模擬
    🎯 情境：你和朋友都想要《JavaScript 入門》
    📄 方案一：影印書（基本型別）

    圖書館幫你們各印一本完整的書：

    • 你在第10頁寫筆記 ➜ 只有你的書有筆記
    • 朋友在第15頁畫重點 ➜ 只有朋友的書有重點
    • 你們各自擁有獨立的書

yourBook += " + 我的筆記";    // 你加筆記
console.log(friendBook);     // "JavaScript入門" (朋友的書沒變)
    🎫 方案二：借書證（參考型別）

    圖書館給你們借書證「A架第3層第5本」：

    • 你去A架拿書，在第10頁寫筆記
    • 朋友後來也去A架拿書 ➜ 看到你的筆記！
    • 因為你們操作的是同一本書

console.log(friendCard.notes);   // ["我的筆記"] (朋友也看得到!)

    🔍 關鍵差異：

    • 影印書（基本型別）：獨立副本，互不影響，佔用更多空間
    • 借書證（參考型別）：共用資源，互相影響，節省空間
    • 記憶體概念：影印書需要複製整本書，借書證只是一張小紙條
    💡 圖書館管理員的智慧

    為什麼圖書館要用借書證而不是影印書？

    ✅ 借書證的優點：
    • 節省空間（不用印很多本）
    • 更新方便（只需更新一本書）
    • 資源共享（大家看的是最新版）
    ⚠️ 借書證的缺點：
    • 容易互相影響（筆記會被看到）
    • 需要小心操作（避免弄壞書）
    • 比較複雜（需要記住位置）

A7. AJAX 與非同步處理

  • A7. AJAX 與非同步處理
    • AJAX 概念： Asynchronous JavaScript and XML，實現網頁與伺服器的非同步資料交換。
    • XMLHttpRequest： 原生 JavaScript 的 AJAX 實作方法，支援各種 HTTP 請求。
    • Fetch API： 現代化的網路請求 API，使用 Promise 簡化非同步操作。
    • JSON 處理： JavaScript Object Notation，現代網路應用的主要資料交換格式。
    • 錯誤處理： 處理網路錯誤、HTTP 狀態碼和資料驗證。
    什麼是 AJAX？

    AJAX (Asynchronous JavaScript and XML) 是一種在不重新載入整個網頁的情況下，與伺服器交換資料並更新部分網頁內容的技術。

    AJAX 的優勢
    • 提升使用者體驗：頁面不需要重新載入
    • 減少伺服器負載：只傳輸需要的資料
    • 加快載入速度：非同步載入內容
    • 動態互動：即時更新頁面內容
    同步 vs 非同步
    // 同步執行（會阻塞）

B1. jQuery 語法與選擇器
什麼是 jQuery？

jQuery 是一個快速、小巧且功能豐富的 JavaScript 函式庫。它透過易於使用的 API 簡化了 HTML 文件遍歷和操作、事件處理、動畫和 Ajax。

基本語法

jQuery 的基本語法是：$('選擇器').方法()

常用選擇器：
  • $('#id') - 選擇 ID
  • $('.class') - 選擇 類別
  • $('element') - 選擇 HTML 元素
  • $('element.class') - 選擇特定類別的元素
  • $(':first') - 選擇第一個元素
  • $(':even') - 選擇偶數位置的元素
程式碼範例：
// 1. ID 選擇器 - 選擇特定 ID 的元素
$('#myButton').click(function() {
    alert('按鈕被點擊了！');
});

// 2. 類別選擇器 - 選擇所有相同類別的元素
$('.highlight').css('background-color', 'yellow');

// 3. 元素選擇器 - 選擇所有相同標籤的元素
$('p').css('color', 'blue');

// 4. 組合選擇器 - 選擇特定類別的特定元素
$('div.warning').addClass('border-red');

// 5. 偽選擇器 - 選擇第一個元素
$('li:first').text('我是第一個項目');

// 6. 偽選擇器 - 選擇偶數位置的元素（索引從0開始）
$('tr:even').css('background-color', '#f8f9fa');

// 7. 屬性選擇器 - 選擇具有特定屬性的元素
$('input[type="text"]').val('預設值');

// 8. 多重選擇器 - 同時選擇多種元素
$('h1, h2, h3').css('font-family', 'Arial');

// 9. 子元素選擇器 - 選擇直接子元素
$('#menu > li').addClass('nav-item');

// 10. 後代選擇器 - 選擇所有後代元素
$('#container .item').show();

    B4. AJAX 初步認識

    AJAX（Asynchronous JavaScript and XML）允許網頁在不重新載入整個頁面的情況下，與伺服器進行資料交換。jQuery 提供了簡潔的 AJAX 方法。

    jQuery AJAX 方法：
    • $.get() - 發送 GET 請求
    • $.post() - 發送 POST 請求
    • $.ajax() - 完整的 AJAX 設定
    • .load() - 載入外部內容到元素中
    AJAX 程式碼範例：
    // 1. $.get() - 發送 GET 請求
$.get('/api/users', function(data) {
    console.log('獲取到的資料：', data);
    // 處理回傳的資料
}).fail(function() {
    alert('請求失敗！');
});

// 2. $.post() - 發送 POST 請求

    name: '張小明',
    email: 'ming@example.com'
}, function(response) {
    alert('使用者建立成功！');
}).fail(function() {
    alert('建立失敗！');
});

// 3. $.ajax() - 完整的 AJAX 設定
$.ajax({
    url: '/api/data',
    method: 'GET',
    dataType: 'json',

        console.log('成功：', data);

    ⚠️ 安全風險
    • 跨站請求偽造 (CSRF)
      使用 CSRF token 驗證
    • 跨站腳本攻擊 (XSS)

    • 敏感資訊洩露
      避免在前端暴露機密
    • 未授權存取

    • 錯誤處理

    • 超時處理

    • 資料驗證
      驗證回傳的資料格式
    重要：永遠不要只依賴前端驗證，後端必須進行完整的安全性檢查。

A. 網站架構基礎

  • A1. 網站的前後端架構概念（Client/Server）
    網站通常分為「前端」與「後端」兩個部分：
    • 前端（Frontend）：負責網頁在瀏覽器上的顯示與互動，常用技術有 HTML、CSS、JavaScript。
    • 後端（Backend）：負責資料儲存、邏輯處理與伺服器端運算，常用技術有 PHP、Node.js、Python、資料庫等。
    Client/Server 與前端/後端的關係
    • Client（客戶端）= 前端：使用者的瀏覽器就是客戶端，負責發送請求並顯示回應的內容。
    • Server（伺服器端）= 後端：遠端的伺服器負責接收請求、處理邏輯、存取資料庫，然後回傳結果。
    • Client-Server 架構：這是一種分散式系統架構，客戶端與伺服器透過網路進行溝通，各自負責不同的工作。
    完整的網站前後端架構示意圖
    • 使用者在瀏覽器（前端）輸入網址，透過網路請求後端伺服器。
    • 後端伺服器處理請求，必要時與資料庫互動，最後回傳資料給前端顯示。
  • A2. 什麼是 HTML、CSS、JavaScript？
    • HTML（HyperText Markup Language - 超文本標記語言）：用來建立網頁的內容與結構，例如標題、段落、圖片、表單等。
      （例：<h1>標題</h1>、<p>段落</p>）
    • CSS（Cascading Style Sheets - 層疊樣式表）：用來美化網頁的外觀，包括顏色、字型、排版、間距、動畫等。
      （例：color: blue; font-size: 20px;）
    • JavaScript：用來讓網頁具有互動性，例如按鈕點擊、表單驗證、動態內容切換等。
      （例：點擊按鈕顯示提示訊息）
  • A3. 前端工程師 vs 後端工程師 vs 全端工程師
    • 前端工程師（Frontend Developer）：專精於使用者介面開發，負責網頁的視覺呈現與互動體驗。
      主要技能：HTML、CSS、JavaScript、Angular/React/Vue、UI/UX 設計
    • 後端工程師（Backend Developer）：專精於伺服器端邏輯，負責資料處理、API 開發與系統架構。
      主要技能：Node.js/PHP/Python/ASP.NET、資料庫、雲端服務、系統安全
    • 全端工程師（Full-Stack Developer）：同時具備前端與後端開發能力，能獨立完成完整的網站專案。
      主要技能：前端 + 後端技術、專案管理、系統整合
  • A4. 瀏覽器如何解析 HTML
    瀏覽器接收到 HTML 文件時，會採用「邊解析邊載入」的方式處理：
    1. 從上到下逐行解析 HTML：瀏覽器開始讀取 HTML 程式碼，識別各種標籤與屬性。
    2. 邊解析邊建立 DOM 樹：將遇到的 HTML 標籤逐步轉換成文件物件模型（DOM），形成樹狀結構。
    3. 遇到外部資源時立即載入：當解析到 <link>、<script>、<img> 時，瀏覽器會同時發出請求載入 CSS、JavaScript、圖片等資源。
    4. 套用 CSS 樣式：CSS 載入完成後，立即套用到已建立的 DOM 元素上。
    5. 執行 JavaScript：JavaScript 載入完成後執行（可能會暫停 HTML 解析）。
    6. 渲染頁面：根據 DOM 樹和樣式資訊，在螢幕上繪製網頁內容。
    重要觀念：瀏覽器是「邊解析邊載入」，所以 CSS 要放在 <head> 中優先載入，JavaScript 建議放在 </body> 前避免阻塞頁面顯示！

B. HTML結構與語法

  • B1. HTML 基本結構與常用標籤
    HTML 文件有固定的基本結構，每個標籤都有其特定功能：
    HTML 基本結構
    <!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>網頁標題</title>
</head>
<body>
    <!-- 網頁內容放這裡 -->
</body>
</html>
    HTML 基本組成結構詳解

    HTML 文件是由多個不同功能的區塊組成，每個部分都有其特定的作用：

    DOCTYPE 宣告
    <!DOCTYPE html>
    作用：
    • 告訴瀏覽器這是 HTML5 文件
    • 確保瀏覽器使用標準模式解析
    • 必須放在文件的第一行
    • 不寫可能造成瀏覽器進入「怪異模式」
    HTML 根元素
    <html lang="zh-Hant">...</html>
    作用：
    • 包含整個 HTML 文件的根容器
    • lang 屬性指定文件語言（有助於SEO和無障礙設計）
    • 所有其他HTML元素都必須在此標籤內
    • 常用語言代碼：zh-Hant（繁中）、zh-Hans（簡中）、en（英文）
    HEAD 區塊
    <head>...</head>
    功能：
    • 包含網頁的後設資料
    • 內容不會顯示在頁面上
    • 提供給瀏覽器和搜尋引擎的資訊
    常見內容：
    • <title> - 網頁標題
    • <meta> - 後設資料
    • <link> - 外部資源連結
    • <style> - 內部樣式
    • <script> - JavaScript程式
    BODY 區塊
    <body>...</body>
    功能：
    • 包含網頁的可見內容
    • 使用者看到的所有內容都在這裡
    • 瀏覽器會渲染此區塊的內容
    常見內容：
    • 標題、段落、圖片
    • 表單、按鈕、連結
    • 表格、清單
    • 導航選單、頁尾
    完整結構樹狀圖
    HTML 文件
├── <!DOCTYPE html>                    ← 文件類型宣告
└── <html>                            ← 根元素
    ├── <head>                        ← 文件頭部（不可見）
    │   ├── <meta charset="UTF-8">    ← 字元編碼
    │   ├── <meta name="viewport">    ← 視窗設定
    │   ├── <title>網頁標題</title>   ← 瀏覽器標題列
    │   ├── <link> CSS 檔案           ← 外部樣式表
    │   └── <script> JavaScript      ← 程式碼
    └── <body>                        ← 文件主體（可見）
        ├── <header>頁首</header>     ← 頁面頂部
        ├── <nav>導航</nav>           ← 導航選單
        ├── <main>主要內容</main>     ← 主要內容區
        └── <footer>頁尾</footer>     ← 頁面底部
    重要概念
    巢狀關係：
    • HTML 元素可以包含其他元素
    • 形成樹狀的層次結構
    • 子元素必須完全包含在父元素內
    語意化結構：
    • 使用適當的標籤表達內容意義
    • 有助於SEO和無障礙設計
    • 讓程式碼更容易維護和理解
  • B2. HTML 標籤語法結構詳解
    HTML 標籤有固定的語法結構，理解這些基礎概念非常重要：
    標籤語法組成要素
    1. 標籤的基本結構
    <標籤名稱 屬性名稱="屬性值">內容</標籤名稱>
    組成部分：
    • < > - 角括號，標示標籤的開始和結束
    • 標籤名稱 - 定義元素類型（如 h1, p, div）
    • 屬性 - 提供標籤額外的資訊和功能
    • 內容 - 標籤包含的文字或其他標籤
    • </標籤名稱> - 結束標籤，告訴瀏覽器元素結束
    2. 開始標籤（Opening Tag）
    <h1 class="title">
    • 用 < 開始，> 結束
    • 包含標籤名稱和屬性
    • 告訴瀏覽器元素開始
    3. 結束標籤（Closing Tag）
    </h1>
    • 用 </ 開始，> 結束
    • 只包含標籤名稱
    • 告訴瀏覽器元素結束
    屬性與標籤範例
    4. 屬性（Attributes） 
    <img src="圖片路徑" alt="替代文字" width="300">
    屬性規則：
    • 格式：屬性名稱="屬性值"
    • 引號：屬性值建議用雙引號包圍
    • 多個屬性：用空格分隔
    • 大小寫：HTML不區分大小寫，但建議用小寫
    5. 完整標籤範例
    <p class="intro">這是段落</p>
<a href="https://google.com">連結</a>
<img src="photo.jpg" alt="照片">
    6. 自閉合標籤
    <br>        <!-- 換行 -->
<hr>        <!-- 水平線 -->
<img>       <!-- 圖片 -->
<input>     <!-- 輸入框 -->
    無需結束標籤的特殊標籤
    重要提醒
    • 配對原則：每個開始標籤都必須有對應的結束標籤（自閉合標籤除外）
    • 巢狀結構：標籤要正確巢狀，不能交錯 ❌ <b><i>文字</b></i>
    • 屬性值：建議總是使用引號包圍，避免解析錯誤

  • B4. HTML 常見屬性說明
    HTML 屬性（Attributes）為標籤提供額外的資訊和功能：
    屬性分類與用法
    🔧 通用屬性（所有標籤都可使用）
    • id - 唯一識別碼，一個頁面內不可重複
    • class - 樣式類別，可重複使用
    • style - 直接設定 CSS 樣式
    • title - 提示文字（滑鼠懸停時顯示）
    • lang - 設定語言（如 zh-Hant、en）
    • data-* - 自訂資料屬性
    • hidden - 隱藏元素
    • tabindex - 設定鍵盤切換順序
    標籤專用屬性範例
    📸 圖片標籤屬性
    <img src="圖片路徑.jpg" alt="替代文字" width="300" height="200" title="圖片說明">
    • src - 圖片來源路徑（必填）
    • alt - 替代文字，無障礙重要屬性（必填）
    • width/height - 圖片寬度/高度
    🔗 連結標籤屬性
    <a href="https://example.com" target="_blank">外部連結</a>
<a href="#section1">內部錨點</a>
<a href="mailto:test@example.com">發送郵件</a>
    • href - 連結目標（網址、錨點、郵件等）
    • target - 開啟方式（_blank 新視窗、_self 當前視窗）
    📝 表單相關屬性
    <input type="text" id="username" name="username" placeholder="請輸入使用者名稱" required>
<label for="username">使用者名稱：</label>
    • type - 輸入類型（text、email、password、number等）
    • name - 表單欄位名稱（提交時的識別）
    • placeholder - 提示文字
    • required - 必填欄位
    • for - label 標籤對應的 input id
    重要提醒與補充
    屬性使用重點：
    • id 和 class 不只給 CSS 用，也可用於 JavaScript 操作
    • 本課程範例中的 class 屬性是給 Bootstrap 框架用的
    • 之後會詳細教學，現在可以先不用理會
    語意化標籤重要性：
    • 使用適當的標籤表達內容意義
    • 有助於SEO和無障礙設計
    • 讓程式碼更容易維護和理解
    語意化標籤的重要性

    語意化標籤（Semantic Tags）不只是用來分割內容，更重要的是表達內容的語意和結構意義：

    🏗️ 結構語意化標籤
    • <header> - 頁首或區塊標題區域
    • <main> - 主要內容（一個頁面只有一個）
    • <footer> - 頁尾或區塊底部資訊
    • <nav> - 導覽連結區域
    • <section> - 內容章節
    • <article> - 獨立的文章內容
    • <aside> - 側邊欄或相關內容
    💡 語意化的好處
    • SEO 優化：搜尋引擎更容易理解內容結構
    • 無障礙網頁：螢幕閱讀器能正確解讀頁面
    • 程式碼可讀性：開發者更容易維護
    • 語意清晰：每個標籤都有明確的意義
    語意化標籤對比範例

  <div class="logo">網站標題</div>

    <div>關於我們</div>
  </div>
</div>
<div class="content">
  <div class="post">
    <div class="title">文章標題</div>
    <div class="text">文章內容...</div>
  </div>
</div>
<div class="footer">版權資訊</div>
    ✅ 語意化寫法
    <header>
  <h1>網站標題</h1>
  <nav>
    <a href="#">首頁</a>
    <a href="#">關於我們</a>

    重點提醒：語意化標籤讓網頁「有意義」，不只是「有樣子」！
    div 和 span 的關鍵差異
    📦 <div> - 區塊容器
    • 顯示方式：區塊元素 (block)
    • 換行：會自動換行，佔滿整行
    • 用途：大範圍版面分組
    • 內容：可包含其他區塊或行內元素
    • 樣式：有預設上下間距
    📝 <span> - 行內容器
    • 顯示方式：行內元素 (inline)
    • 換行：不會換行，在同一行內
    • 用途：文字中的局部樣式
    • 內容：只能包含行內元素或文字
    • 樣式：無預設間距
    對比範例
    <!-- div 範例：每個都會換行 -->
<div>第一個區塊</div>
<div>第二個區塊</div>
<div>第三個區塊</div>

<!-- span 範例：都在同一行 -->
<span>第一個</span><span>第二個</span><span>第三個</span>

<!-- 混合使用範例 -->
<div class="card">
    <h3>商品資訊</h3>
    <p>商品名稱：<span class="product-name">iPhone 15</span></p>
    <p>價格：<span class="price">$999</span></p>
</div>
    記憶口訣：div 分區塊，span 改樣式！

  • B5. HTML 表格標籤介紹
    HTML 表格用來呈現結構化資料，如統計資料、價格表、時間表等，具有行與列的二維結構：
    表格基本結構標籤
    • <table> - 表格容器
    • <thead> - 表格標題區域
    • <tbody> - 表格內容區域
    • <tfoot> - 表格底部區域
    • <tr> - 表格列（table row）
    • <th> - 表格標題欄位（table header）
    • <td> - 表格資料欄位（table data）
    • <caption> - 表格標題說明
    基本表格範例
    實際顯示效果比對：
    沒有樣式的表格
    學生成績表
    姓名 數學 英語 平均
    王小明 85 92 88.5
    李小華 78 86 82
    有樣式的表格
    學生成績表
    姓名 數學 英語 平均
    王小明 85 92 88.5
    李小華 78 86 82
    樣式差異說明：
    • 沒有樣式：使用基本的 HTML 表格，僅有 border="1" 屬性
    • 有樣式：使用 Bootstrap 的 table、table-bordered、table-light 等 CSS 類別
    • 視覺效果：有樣式的表格具有更好的間距、顏色對比和整體美觀性
    HTML 程式碼：
    <table border="1">
  <caption>學生成績表</caption>
  <thead>
    <tr>
      <th>姓名</th>
      <th>數學</th>
      <th>英語</th>
      <th>平均</th>
    </tr>

      CSS（Cascading Style Sheets）的「層疊」是其核心概念，了解優先權規則可以幫助你更好地控制樣式的套用。

      CSS 層疊原理

      當多個 CSS 規則作用於同一個元素時，瀏覽器會根據以下順序決定最終的樣式：

      層疊順序（重要性排序）
      🔴 最高優先權
      1. !important 宣告
      2. 內聯樣式（style 屬性）
      3. 內部樣式表（<style> 標籤）
      4. 外部樣式表（.css 檔案）
      🟢 最低優先權
      1. 瀏覽器預設樣式
      注意：相同權重時，後出現的規則會覆蓋前面的規則
      選擇器特異性（Specificity）計算

      當多個選擇器作用於同一元素時，特異性高的規則會勝出：

      選擇器類型 權重值 範例 計算方式
      內聯樣式 1000 style="color: red;" 1-0-0-0
      ID 選擇器 100 #header 0-1-0-0
      類別選擇器 10 .button 0-0-1-0
      屬性選擇器 10 [type="text"] 0-0-1-0
      偽類選擇器 10 :hover 0-0-1-0
      元素選擇器 1 div 0-0-0-1
      偽元素選擇器 1 ::before 0-0-0-1
      🧮 特異性計算範例
      /* 特異性計算範例 */
div p                    /* 0-0-0-2 = 2 */
.article p               /* 0-0-1-1 = 11 */
#content .article p      /* 0-1-1-1 = 111 */
div#content .article p   /* 0-1-1-2 = 112 */

/* 複雜範例 */
nav ul li a:hover        /* 0-0-1-4 = 14 */
.nav .menu-item:hover    /* 0-0-2-0 = 20 */
#nav .menu-item          /* 0-1-1-0 = 110 */
      !important 的使用與注意事項

      !important 宣告會覆蓋其他所有樣式，但應謹慎使用：

      ✅ 適當使用場景
      • 覆蓋第三方樣式：無法修改的外部 CSS
      • 實用程式類別：確保工具類別生效
      • 臨時修正：緊急問題的快速解決
      • 樣式重置：強制重置某些屬性
      ❌ 應避免的情況
      • 日常開發：不應作為常規解決方案
      • 維護困難：破壞層疊規則邏輯
      • 調試困難：難以追蹤樣式來源
      • 擴展性差：後續修改變得複雜
      🔥 實際範例
      /* 不好的做法 */
.button {
    background-color: blue !important;
    color: white !important;
}

/* 更好的做法：提高選擇器特異性 */
.page .content .button {
    background-color: blue;
    color: white;
}

/* 或使用更具體的類別 */
.button.primary {
    background-color: blue;
    color: white;
}

      ✅ 推薦做法
      • 遵循 BEM 命名：減少嵌套依賴
      • 保持低特異性：儘量使用單一類別
      • 邏輯分層：基礎樣式 → 元件樣式 → 工具樣式

      • 過度嵌套：避免超過 3 層選擇器

      盒模型基礎概念

      CSS 盒模型（Box Model）是理解網頁版面配置的核心概念。每個 HTML 元素在瀏覽器中都被視為一個矩形的「盒子」，這個盒子由內到外包含四個不同的區域，彼此環環相扣，共同決定了元素的最終外觀與佔用空間。

      Margin (外邊距) - 透明區域
      Border (邊框)
      Padding (內邊距)
      Content
      內容區域
      1Content 內容區域
      • 定義：實際顯示文字、圖片等內容的核心區域
      • 控制：由 width 和 height 屬性決定大小
      • 特性：是盒模型的最內層，所有其他區域都圍繞著它
      • 重要性：決定了元素能夠容納多少實際內容
      .content-demo {
        width: 200px;
        height: 100px;
      }
      2Padding 內邊距
      • 定義：內容與邊框之間的緩衝空間
      • 作用：增加元素內部的留白，提升可讀性
      • 視覺：會延伸背景色或背景圖片
      • 設定：可分別設定上右下左四個方向
      .padding-demo {
        padding: 20px;
        /* 或分別設定 */
        padding: 10px 15px 10px 15px;
      }
      3Border 邊框

      • 屬性：可設定粗細、樣式、顏色
      • 影響：佔據實際空間，影響元素總尺寸
      • 樣式：solid、dashed、dotted 等多種樣式
      .border-demo {
        border: 2px solid #007bff;
        /* 或分別設定 */
        border-width: 1px 2px 1px 2px;

      5.2 盒模型計算規則與 Box-sizing 屬性對比

      理解盒模型的計算規則是掌握網頁版面配置的關鍵。CSS 提供了兩種不同的盒模型計算方式：content-box（瀏覽器預設）和 border-box。掌握兩者的差異對於精確控制版面配置至關重要。

      Content-box（預設模式）
      瀏覽器預設：box-sizing: content-box;
      📐 計算公式
      元素總寬度 =
      width + padding-left + padding-right + border-left + border-right
      元素總高度 =
      height + padding-top + padding-bottom + border-top + border-bottom
      ✨ 特點
      • width/height 只控制內容區域
      • padding 和 border 會額外增加元素尺寸
      • 需要手動計算總尺寸
      • 適合內容導向的設計
      💡 使用場景
      • 文字內容區域設計
      • 需要精確控制內容大小
      • 傳統網頁設計模式
      Border-box（現代推薦）
      需要設定：box-sizing: border-box;
      📐 計算公式
      元素總寬度 =
      width（包含 padding 和 border）
      元素總高度 =
      height（包含 padding 和 border）
      ✨ 特點
      • width/height 就是元素總尺寸
      • padding 和 border 向內壓縮內容區域
      • 尺寸計算更直觀
      • 適合版面配置設計
      💡 使用場景
      • 響應式網頁設計
      • Grid 和 Flexbox 佈局
      • 需要精確控制元素尺寸
      • 現代 CSS 框架
      實際範例對比：相同設定下的不同結果

      5.4 Margin 與 Padding 的使用技巧

      Margin（外邊距）和 Padding（內邊距）是盒模型中最常使用的屬性，掌握它們的特性與技巧對於精確控制版面配置至關重要。本節將深入探討這兩個屬性的進階應用。

      Margin 外邊距
      定義：元素與其他元素之間的距離
      🔍 特性
      • 透明背景：不受元素背景色影響
      • 可合併：垂直方向會發生 margin 合併
      • 可為負值：負 margin 可縮小間距
      • 不影響點擊區域：margin 區域無法點擊
      📐 語法
      margin: 10px; /* 四邊相同 */
      margin: 10px 20px; /* 上下 左右 */
      margin: 10px 20px 30px 40px; /* 上 右 下 左 */
      Padding 內邊距
      定義：元素內容與邊框之間的距離
      🔍 特性
      • 繼承背景：顯示元素的背景色
      • 不會合併：各元素獨立計算
      • 只能為正值：不接受負值
      • 擴大點擊區域：padding 區域可點擊
      📐 語法
      padding: 15px; /* 四邊相同 */
      padding: 15px 25px; /* 上下 左右 */
      padding: 10px 20px 30px 40px; /* 上 右 下 左 */

      灰色虛線框顯示 margin 透明區域
      Padding 效果
      Padding: 20px
      內容區 + 內邊距
      綠色區域包含 padding，顯示背景色
      重要概念：Margin 合併（Margin Collapse）

      垂直方向上相鄰元素的 margin 會合併，取較大值而非相加。這是 CSS 的特殊行為，需要特別注意。

      🔍 合併情況
      • 相鄰兄弟元素：上下 margin 會合併
      • 父子元素：父元素與第一個/最後一個子元素
      • 空元素：自身的上下 margin 會合併
      • 只在垂直方向：水平方向不會合併
      實際範例與解決方案
      ❌ 問題：Margin 合併現象
      .box1 { margin-bottom: 30px; }
.box2 { margin-top: 20px; }
      元素 1 (margin-bottom: 30px)
      元素 2 (margin-top: 20px)
      實際間距：30px（合併取較大值）
      合併問題：
      • 預期間距：30px + 20px = 50px
      • 實際間距：30px（只取較大值）
      • 造成佈局不如預期
      ✅ 解決方案 1：只使用單方向 margin
      .box1 { margin-bottom: 50px; }
.box2 { margin-top: 0; }
      元素 1 (margin-bottom: 50px)
      元素 2 (margin-top: 0)
      實際間距：50px（符合預期）
      最佳實踐：
      • 統一使用 margin-bottom 或 margin-top
      • 避免同時設定上下 margin
      • 佈局更可預測
      ✅ 解決方案 2：使用父容器 Padding 建立間距
      💡 核心概念
      每個元素分別包在不同父容器內，用父容器的 padding 建立間距，避免 margin 合併。 
      /* 每個元素包在不同容器內 */
.container-1 {
  padding-bottom: 25px; /* 第一個元素的下方間距 */
  background: #e8f5e8;
}
.container-2 {
  padding-top: 25px;    /* 第二個元素的上方間距 */
  background: #fff3e0;
}
.box { margin: 0; }
      元素 1 (margin: 0)
      容器 1：padding-bottom: 25px
      容器 2：padding-top: 25px
      元素 2 (margin: 0)
      總間距：50px（25px + 25px，無合併問題）
      Padding 間距策略：
      • 關鍵：每個元素分別包在不同父容器
      • 第一個容器用 padding-bottom
      • 第二個容器用 padding-top
      • 子元素設定 margin: 0
      • 完全隔離，無合併問題
      • 間距 = padding-bottom + padding-top
      HTML 結構對比
      ❌ 錯誤結構（會合併）
      <div class="parent">
  <div class="box1">元素 1</div>
  <div class="box2">元素 2</div>
</div>

.box1 { margin-bottom: 30px; }
.box2 { margin-top: 20px; }
      兩個元素在同一父容器內，會發生 margin 合併
      ✅ 正確結構（用 padding 避免）
      <div class="container-1">
  <div class="box">元素 1</div>
</div>

  <div class="box">元素 2</div>
</div>

.container-2 { padding-top: 25px; }
.box { margin: 0; }
      每個元素在不同父容器內，用 padding 建立間距
      ✅ 解決方案 3：現代化間距控制方法
      🚀 Flexbox Gap（最推薦）
      .flex-container {
  display: flex;
  flex-direction: column;
  gap: 30px; /* 現代化間距方案 */
}
.flex-item { margin: 0; }

      Flex 項目 2
      Flex 項目 3

      6.1 Display 顯示模式

      Display 屬性是 CSS 中最重要的屬性之一，它決定了元素如何顯示以及如何與其他元素互動。理解不同 display 值的特性，是掌握網頁佈局的關鍵基礎。

      每個 HTML 元素都有預設的 display 值，但我們可以透過 CSS 改變它：

      • 語意化優先：先選擇合適的 HTML 元素
      • 漸進增強：基本功能不依賴 CSS
      • 現代佈局：新專案優先考慮 Flexbox/Grid
      • 響應式友善：考慮不同螢幕尺寸的顯示
      • 可訪問性：確保隱藏內容不影響螢幕閱讀器
      ❌ 避免做法
      • 過度依賴：不要只用 div + display 解決所有問題
      • 混亂結構：避免不必要的 display 改變
      • 兼容性問題：注意舊瀏覽器支援度
      • 可維護性：避免複雜的 table 顯示模式
      • 效能考量：頻繁切換 display 會影響效能
      💡 選擇建議總結
      • Block：頁面結構、內容區塊、獨立元件
      • Inline：文字樣式、小標記、簡單連結
      • Inline-block：按鈕、小卡片、工具列項目
      • None：條件顯示、響應式隱藏、互動控制
      • Flex/Grid：現代佈局的首選方案

      6.3 Z-index 層級控制

      Z-index 控制元素在 Z 軸（垂直於螢幕）上的層疊順序，是實現彈出視窗、下拉選單、模態框等覆蓋效果的關鍵技術。掌握層疊上下文的規則，能避免常見的層級顯示問題。

      Z-index 層級基本概念

      Z-index 決定元素在 Z 軸上的層疊順序，只對定位元素（非 static）生效：

      📚 層級規則
      • z-index: auto - 預設值，不建立新層疊上下文
      • z-index: 整數 - 正數在上，負數在下
      • 數值越大越在上方
      • 只對定位元素有效
      🎯 層疊上下文
      • 根元素：html 是根層疊上下文
      • 定位元素：position 非 static 且 z-index 非 auto
      • 其他觸發：opacity、transform、filter 等
      • 獨立管理：每個上下文內部獨立比較

      • 避免極值：不要使用過大或過小的數值
      • 組件規範：為不同組件定義固定範圍

      Float 浮動是早期 CSS 佈局的重要技術，雖然現在多被 Flexbox 和 Grid 取代，但理解 Float 的運作原理對於維護舊程式碼和實現特殊效果（如文字環繞）仍然重要。掌握浮動清除技巧能避免常見的佈局問題。

      Float 浮動基本概念

      Float 讓元素脫離正常文件流，向左或向右浮動到父容器邊緣，其他內容會環繞浮動元素：

      🏊 浮動行為
      • float: left - 向左浮動
      • float: right - 向右浮動
      • float: none - 不浮動（預設）
      • 脫離文件流但保留對其他元素的影響
      📦 浮動特性
      • 塊級化：浮動元素變為塊級行為
      • 收縮包裹：寬度收縮到內容大小
      • 層級提升：位於普通元素之上
      • 影響佈局：其他元素會環繞浮動元素

      6.6 現代佈局方法預覽
      現代佈局發展概述

      隨著網頁設計需求的複雜化，傳統的佈局方法（如 float、position）已無法有效滿足現代網頁的靈活性要求。 CSS3 引入了兩大革命性的佈局技術：Flexbox（彈性盒模型）和 CSS Grid（網格佈局）， 為開發者提供了更直觀、更強大的佈局工具。

      佈局演進歷程： 表格佈局 → Float 佈局 → Position 定位 → Flexbox → CSS Grid → 混合應用
      6.6.1 Flexbox 彈性盒模型簡介
      核心概念

      Flexbox（Flexible Box Layout）是一維佈局方法，專門處理容器內元素的排列、對齊和空間分配。 它基於主軸（main axis）和交叉軸（cross axis）的概念進行佈局。

      主要特點：
      • 一維佈局（行或列）
      • 自動空間分配
      • 靈活的對齊方式
      • 元素順序控制
      適用場景：
      • 導航選單佈局
      • 卡片排列
      • 垂直/水平置中
      • 等高欄位
      基本語法展示 
      /* 建立彈性容器 */
.flex-container {
    display: flex;
    justify-content: center;    /* 主軸對齊 */
    align-items: center;        /* 交叉軸對齊 */
    gap: 20px;                  /* 元素間距 */
}

/* 彈性子元素 */
.flex-item {
    flex: 1;                    /* 彈性增長 */
    min-width: 200px;          /* 最小寬度 */
}

      6.6.2 CSS Grid 網格佈局簡介
      核心概念

      CSS Grid 是二維佈局系統，可同時控制行（rows）和列（columns）， 提供了前所未有的佈局控制能力，特別適合複雜的頁面佈局。

      主要特點：
      • 二維佈局（行+列）
      • 精確的區域控制
      • 重疊與層次
      • 響應式友好
      適用場景：
      • 整體頁面佈局
      • 複雜的內容網格
      • 雜誌式佈局
      • 儀表板介面
      基本語法展示 
      /* 建立網格容器 */
.grid-container {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;  /* 3欄佈局 */
    grid-template-rows: auto 1fr auto;   /* 3列佈局 */
    gap: 20px;                           /* 網格間距 */
    height: 100vh;                       /* 全螢幕高度 */
}

/* 網格區域命名 */
.grid-container {
    grid-template-areas: 
        "header header header"
        "sidebar main aside"
        "footer footer footer";
}

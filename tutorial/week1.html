A. 網站架構基礎

  • A1. 網站的前後端架構概念（Client/Server）
    網站通常分為「前端」與「後端」兩個部分：
    • 前端（Frontend）：負責網頁在瀏覽器上的顯示與互動，常用技術有 HTML、CSS、JavaScript。
    • 後端（Backend）：負責資料儲存、邏輯處理與伺服器端運算，常用技術有 PHP、Node.js、Python、資料庫等。
    Client/Server 與前端/後端的關係
    • Client（客戶端）= 前端：使用者的瀏覽器就是客戶端，負責發送請求並顯示回應的內容。
    • Server（伺服器端）= 後端：遠端的伺服器負責接收請求、處理邏輯、存取資料庫，然後回傳結果。
    • Client-Server 架構：這是一種分散式系統架構，客戶端與伺服器透過網路進行溝通，各自負責不同的工作。
    完整的網站前後端架構示意圖
    • 使用者在瀏覽器（前端）輸入網址，透過網路請求後端伺服器。
    • 後端伺服器處理請求，必要時與資料庫互動，最後回傳資料給前端顯示。
  • A2. 什麼是 HTML、CSS、JavaScript？
    • HTML（HyperText Markup Language - 超文本標記語言）：用來建立網頁的內容與結構，例如標題、段落、圖片、表單等。
      （例：<h1>標題</h1>、<p>段落</p>）
    • CSS（Cascading Style Sheets - 層疊樣式表）：用來美化網頁的外觀，包括顏色、字型、排版、間距、動畫等。
      （例：color: blue; font-size: 20px;）
    • JavaScript：用來讓網頁具有互動性，例如按鈕點擊、表單驗證、動態內容切換等。
      （例：點擊按鈕顯示提示訊息）
  • A3. 前端工程師 vs 後端工程師 vs 全端工程師
    • 前端工程師（Frontend Developer）：專精於使用者介面開發，負責網頁的視覺呈現與互動體驗。
      主要技能：HTML、CSS、JavaScript、Angular/React/Vue、UI/UX 設計
    • 後端工程師（Backend Developer）：專精於伺服器端邏輯，負責資料處理、API 開發與系統架構。
      主要技能：Node.js/PHP/Python/ASP.NET、資料庫、雲端服務、系統安全
    • 全端工程師（Full-Stack Developer）：同時具備前端與後端開發能力，能獨立完成完整的網站專案。
      主要技能：前端 + 後端技術、專案管理、系統整合
  • A4. 瀏覽器如何解析 HTML
    瀏覽器接收到 HTML 文件時，會採用「邊解析邊載入」的方式處理：
    1. 從上到下逐行解析 HTML：瀏覽器開始讀取 HTML 程式碼，識別各種標籤與屬性。
    2. 邊解析邊建立 DOM 樹：將遇到的 HTML 標籤逐步轉換成文件物件模型（DOM），形成樹狀結構。
    3. 遇到外部資源時立即載入：當解析到 <link>、<script>、<img> 時，瀏覽器會同時發出請求載入 CSS、JavaScript、圖片等資源。
    4. 套用 CSS 樣式：CSS 載入完成後，立即套用到已建立的 DOM 元素上。
    5. 執行 JavaScript：JavaScript 載入完成後執行（可能會暫停 HTML 解析）。
    6. 渲染頁面：根據 DOM 樹和樣式資訊，在螢幕上繪製網頁內容。
    重要觀念：瀏覽器是「邊解析邊載入」，所以 CSS 要放在 <head> 中優先載入，JavaScript 建議放在 </body> 前避免阻塞頁面顯示！

B. HTML結構與語法

  • B1. HTML 基本結構與常用標籤
    HTML 文件有固定的基本結構，每個標籤都有其特定功能：
    HTML 基本結構
    <!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>網頁標題</title>
</head>
<body>
    <!-- 網頁內容放這裡 -->
</body>
</html>
    HTML 基本組成結構詳解

    HTML 文件是由多個不同功能的區塊組成，每個部分都有其特定的作用：

    DOCTYPE 宣告
    <!DOCTYPE html>
    作用：
    • 告訴瀏覽器這是 HTML5 文件
    • 確保瀏覽器使用標準模式解析
    • 必須放在文件的第一行
    • 不寫可能造成瀏覽器進入「怪異模式」
    HTML 根元素
    <html lang="zh-Hant">...</html>
    作用：
    • 包含整個 HTML 文件的根容器
    • lang 屬性指定文件語言（有助於SEO和無障礙設計）
    • 所有其他HTML元素都必須在此標籤內
    • 常用語言代碼：zh-Hant（繁中）、zh-Hans（簡中）、en（英文）
    HEAD 區塊
    <head>...</head>
    功能：
    • 包含網頁的後設資料
    • 內容不會顯示在頁面上
    • 提供給瀏覽器和搜尋引擎的資訊
    常見內容：
    • <title> - 網頁標題
    • <meta> - 後設資料
    • <link> - 外部資源連結
    • <style> - 內部樣式
    • <script> - JavaScript程式
    BODY 區塊
    <body>...</body>
    功能：
    • 包含網頁的可見內容
    • 使用者看到的所有內容都在這裡
    • 瀏覽器會渲染此區塊的內容
    常見內容：
    • 標題、段落、圖片
    • 表單、按鈕、連結
    • 表格、清單
    • 導航選單、頁尾
    完整結構樹狀圖
    HTML 文件
├── <!DOCTYPE html>                    ← 文件類型宣告
└── <html>                            ← 根元素
    ├── <head>                        ← 文件頭部（不可見）
    │   ├── <meta charset="UTF-8">    ← 字元編碼
    │   ├── <meta name="viewport">    ← 視窗設定
    │   ├── <title>網頁標題</title>   ← 瀏覽器標題列
    │   ├── <link> CSS 檔案           ← 外部樣式表
    │   └── <script> JavaScript      ← 程式碼
    └── <body>                        ← 文件主體（可見）
        ├── <header>頁首</header>     ← 頁面頂部
        ├── <nav>導航</nav>           ← 導航選單
        ├── <main>主要內容</main>     ← 主要內容區
        └── <footer>頁尾</footer>     ← 頁面底部
    重要概念
    巢狀關係：
    • HTML 元素可以包含其他元素
    • 形成樹狀的層次結構
    • 子元素必須完全包含在父元素內
    語意化結構：
    • 使用適當的標籤表達內容意義
    • 有助於SEO和無障礙設計
    • 讓程式碼更容易維護和理解
  • B2. HTML 標籤語法結構詳解
    HTML 標籤有固定的語法結構，理解這些基礎概念非常重要：
    標籤語法組成要素
    1. 標籤的基本結構
    <標籤名稱 屬性名稱="屬性值">內容</標籤名稱>
    組成部分：
    • < > - 角括號，標示標籤的開始和結束
    • 標籤名稱 - 定義元素類型（如 h1, p, div）
    • 屬性 - 提供標籤額外的資訊和功能
    • 內容 - 標籤包含的文字或其他標籤
    • </標籤名稱> - 結束標籤，告訴瀏覽器元素結束
    2. 開始標籤（Opening Tag）
    <h1 class="title">
    • 用 < 開始，> 結束
    • 包含標籤名稱和屬性
    • 告訴瀏覽器元素開始
    3. 結束標籤（Closing Tag）
    </h1>
    • 用 </ 開始，> 結束
    • 只包含標籤名稱
    • 告訴瀏覽器元素結束
    屬性與標籤範例
    4. 屬性（Attributes） 
    <img src="圖片路徑" alt="替代文字" width="300">
    屬性規則：
    • 格式：屬性名稱="屬性值"
    • 引號：屬性值建議用雙引號包圍
    • 多個屬性：用空格分隔
    • 大小寫：HTML不區分大小寫，但建議用小寫
    5. 完整標籤範例
    <p class="intro">這是段落</p>
<a href="https://google.com">連結</a>
<img src="photo.jpg" alt="照片">
    6. 自閉合標籤
    <br>        <!-- 換行 -->
<hr>        <!-- 水平線 -->
<img>       <!-- 圖片 -->
<input>     <!-- 輸入框 -->
    無需結束標籤的特殊標籤
    重要提醒
    • 配對原則：每個開始標籤都必須有對應的結束標籤（自閉合標籤除外）
    • 巢狀結構：標籤要正確巢狀，不能交錯 ❌ <b><i>文字</b></i>
    • 屬性值：建議總是使用引號包圍，避免解析錯誤

  • B4. HTML 常見屬性說明
    HTML 屬性（Attributes）為標籤提供額外的資訊和功能：
    屬性分類與用法
    🔧 通用屬性（所有標籤都可使用）
    • id - 唯一識別碼，一個頁面內不可重複
    • class - 樣式類別，可重複使用
    • style - 直接設定 CSS 樣式
    • title - 提示文字（滑鼠懸停時顯示）
    • lang - 設定語言（如 zh-Hant、en）
    • data-* - 自訂資料屬性
    • hidden - 隱藏元素
    • tabindex - 設定鍵盤切換順序
    標籤專用屬性範例
    📸 圖片標籤屬性
    <img src="圖片路徑.jpg" alt="替代文字" width="300" height="200" title="圖片說明">
    • src - 圖片來源路徑（必填）
    • alt - 替代文字，無障礙重要屬性（必填）
    • width/height - 圖片寬度/高度
    🔗 連結標籤屬性
    <a href="https://example.com" target="_blank">外部連結</a>
<a href="#section1">內部錨點</a>
<a href="mailto:test@example.com">發送郵件</a>
    • href - 連結目標（網址、錨點、郵件等）
    • target - 開啟方式（_blank 新視窗、_self 當前視窗）
    📝 表單相關屬性
    <input type="text" id="username" name="username" placeholder="請輸入使用者名稱" required>
<label for="username">使用者名稱：</label>
    • type - 輸入類型（text、email、password、number等）
    • name - 表單欄位名稱（提交時的識別）
    • placeholder - 提示文字
    • required - 必填欄位
    • for - label 標籤對應的 input id
    重要提醒與補充
    屬性使用重點：
    • id 和 class 不只給 CSS 用，也可用於 JavaScript 操作
    • 本課程範例中的 class 屬性是給 Bootstrap 框架用的
    • 之後會詳細教學，現在可以先不用理會
    語意化標籤重要性：
    • 使用適當的標籤表達內容意義
    • 有助於SEO和無障礙設計
    • 讓程式碼更容易維護和理解
    語意化標籤的重要性

    語意化標籤（Semantic Tags）不只是用來分割內容，更重要的是表達內容的語意和結構意義：

    🏗️ 結構語意化標籤
    • <header> - 頁首或區塊標題區域
    • <main> - 主要內容（一個頁面只有一個）
    • <footer> - 頁尾或區塊底部資訊
    • <nav> - 導覽連結區域
    • <section> - 內容章節
    • <article> - 獨立的文章內容
    • <aside> - 側邊欄或相關內容
    💡 語意化的好處
    • SEO 優化：搜尋引擎更容易理解內容結構
    • 無障礙網頁：螢幕閱讀器能正確解讀頁面
    • 程式碼可讀性：開發者更容易維護
    • 語意清晰：每個標籤都有明確的意義
    語意化標籤對比範例
    ❌ 非語意化寫法

  <div class="logo">網站標題</div>
  <div class="menu">

    <div>關於我們</div>
  </div>
</div>
<div class="content">
  <div class="post">
    <div class="title">文章標題</div>
    <div class="text">文章內容...</div>
  </div>
</div>
<div class="footer">版權資訊</div>
    ✅ 語意化寫法
    <header>
  <h1>網站標題</h1>
  <nav>
    <a href="#">首頁</a>
    <a href="#">關於我們</a>

    重點提醒：語意化標籤讓網頁「有意義」，不只是「有樣子」！
    div 和 span 的關鍵差異
    📦 <div> - 區塊容器
    • 顯示方式：區塊元素 (block)
    • 換行：會自動換行，佔滿整行
    • 用途：大範圍版面分組
    • 內容：可包含其他區塊或行內元素
    • 樣式：有預設上下間距
    📝 <span> - 行內容器
    • 顯示方式：行內元素 (inline)
    • 換行：不會換行，在同一行內
    • 用途：文字中的局部樣式
    • 內容：只能包含行內元素或文字
    • 樣式：無預設間距
    對比範例
    <!-- div 範例：每個都會換行 -->
<div>第一個區塊</div>
<div>第二個區塊</div>
<div>第三個區塊</div>

<!-- span 範例：都在同一行 -->
<span>第一個</span><span>第二個</span><span>第三個</span>

<!-- 混合使用範例 -->
<div class="card">
    <h3>商品資訊</h3>
    <p>商品名稱：<span class="product-name">iPhone 15</span></p>
    <p>價格：<span class="price">$999</span></p>
</div>
    記憶口訣：div 分區塊，span 改樣式！

  • B5. HTML 表格標籤介紹
    HTML 表格用來呈現結構化資料，如統計資料、價格表、時間表等，具有行與列的二維結構：
    表格基本結構標籤
    • <table> - 表格容器
    • <thead> - 表格標題區域
    • <tbody> - 表格內容區域
    • <tfoot> - 表格底部區域
    • <tr> - 表格列（table row）
    • <th> - 表格標題欄位（table header）
    • <td> - 表格資料欄位（table data）
    • <caption> - 表格標題說明
    基本表格範例
    實際顯示效果比對：
    沒有樣式的表格
    學生成績表
    姓名 數學 英語 平均
    王小明 85 92 88.5
    李小華 78 86 82
    有樣式的表格
    學生成績表
    姓名 數學 英語 平均
    王小明 85 92 88.5
    李小華 78 86 82
    樣式差異說明：
    • 沒有樣式：使用基本的 HTML 表格，僅有 border="1" 屬性
    • 有樣式：使用 Bootstrap 的 table、table-bordered、table-light 等 CSS 類別
    • 視覺效果：有樣式的表格具有更好的間距、顏色對比和整體美觀性
    HTML 程式碼：
    <table border="1">
  <caption>學生成績表</caption>
  <thead>
    <tr>
      <th>姓名</th>
      <th>數學</th>
      <th>英語</th>
      <th>平均</th>
    </tr>

      4.5 Border vs Outline 差異對比
      基本差異說明
      Border (邊框)
      • 佔據空間：會影響元素的實際大小

      • 圓角支援：支援 border-radius

      • 佈局影響：會推擠周圍元素
      • 主要用途：裝飾、分隔、框定內容

      • 不佔空間：不影響元素大小
      • 覆蓋顯示：浮在元素上方
      • 無圓角：不支援圓角效果
      • 統一樣式：四邊必須相同
      • 無佈局影響：不會推擠其他元素
      • 主要用途：焦點指示、可訪問性

      什麼是 Margin 塌陷？

      當兩個相鄰的塊級元素有垂直 margin 時，它們的間距不會相加，而是取較大的值。這種現象稱為「margin 塌陷」或「margin 合併」。

    background: #007bff;
    padding: 15px;

.box2 {
    margin-top: 20px;     /* 上邊距 20px */
    background: #28a745;
    padding: 15px;
}

/* 實際間距：max(30px, 20px) = 30px */
/* 不是 30px + 20px = 50px */

/* 解決方法 */
.no-collapse {
    padding: 1px 0;       /* 添加 padding */
    /* 或使用 border */
    /* 或使用 overflow: hidden */

      5.3 Margin vs Padding 差異對比
      基本概念差異
      Margin (外邊距)
      • 位置：元素邊框外側的空間
      • 作用：控制元素與其他元素的距離
      • 背景：不顯示元素背景色
      • 透明度：完全透明
      • 點擊：不響應點擊事件
      • 塌陷：會發生 margin collapsing
      Padding (內邊距)
      • 位置：元素邊框內側的空間
      • 作用：控制內容與邊框的距離
      • 背景：顯示元素背景色
      • 透明度：繼承元素背景
      • 點擊：響應點擊事件
      • 塌陷：不會發生塌陷

      Margin 適用場景
      • 元素間距：控制兩個元素之間的距離
      • 版面佈局：創建頁面的整體間距結構

      • 響應式間距：不同螢幕尺寸的間距調整

      • 卡片之間的間距
      • 段落之間的分隔
      • 容器的水平置中
      • 頁面區塊的上下間距

      • 內容間距：文字與邊框的距離
      • 可點擊區域：增加按鈕、連結的點擊範圍
      • 視覺舒適：避免內容貼邊顯示
      • 背景填充：讓背景色有適當的範圍
      • 表單設計：輸入框內的文字間距

    • C6. 盒模型 Box Model
      CSS 盒模型是理解網頁版面配置的基礎概念

      每個 HTML 元素都可以看作一個矩形的盒子，由內到外包含四個區域：內容 (content)、內邊距 (padding)、邊框 (border)、外邊距 (margin)。

      盒模型的四個組成部分
      Content 內容區域
      • 實際顯示文字、圖片等內容的區域
      • 由 width 和 height 屬性控制大小
      • 是盒模型的核心部分
      Padding 內邊距
      • 內容與邊框之間的空間
      • 增加元素內部的留白空間
      • 背景色會延伸到 padding 區域
      Border 邊框
      • 圍繞在 padding 外圍的線條
      • 可設定粗細、樣式、顏色
      • 佔據實際空間，影響元素總大小
      Margin 外邊距
      • 元素與其他元素之間的距離
      • 透明的，不會顯示背景色
      • 用於控制元素間的間距

      盒模型核心概念
      • 標準盒模型 (content-box)：width/height 只包含內容區域
      • 替代盒模型 (border-box)：width/height 包含 padding 和 border

      注意：Float 是較舊的佈局方式，現代開發多使用 Flexbox 或 Grid。但理解 Float 仍有助於維護舊代碼和理解網頁佈局原理。
      1. Float Left 左浮動
      CSS
      .float-left {
                        float: left;
                        width: 150px;
                        height: 100px;
                        background: #e3f2fd;
                        padding: 10px;
                        margin: 0 10px 10px 0;
                        border: 1px solid #2196f3;
                    }
      效果展示
      Float Left

      這段文字會環繞在浮動元素的右側。Float 屬性會讓元素脫離正常的文檔流，並浮動到父容器的左側或右側。其他內容會圍繞浮動元素排列。這是早期網頁佈局常用的技術，用於實現文字環繞圖片的效果。

      5. Float 常見問題與解決方案
      問題：父容器高度塌陷
      解決方案
      /* 方法1: 使用 overflow */
                    .container {
                        overflow: auto; /* 或 hidden */
                    }
                    
                    /* 方法2: 使用 clearfix */
                    .container::after {
                        content: "";
                        display: table;
                        clear: both;
                    }
                    
                    /* 方法3: 父容器也浮動 */
                    .container {
                        float: left;
                        width: 100%;
                    }

      特性 Float Flexbox Grid
      學習難度 中等，需理解清除浮動 簡單，直觀 中等，功能強大
